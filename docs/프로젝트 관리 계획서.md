# 1. 개요
## 1.1 문서의 목적
파일 공유 시스템 개발 프로젝트를 진행하는 데 있어 각 세부 사항의 지표로 삼는 것을 목적으로 한다.

## 1.2 프로젝트 개요
업무 문서 공유를 위한 클라우드 기반 파일 공유 시스템을 개발한다.

주요 기능은 다음과 같다.
- 파일 업로드 및 다운로드
- 폴더 기능
- 파일 공유 및 사용 권한 설정
- 파일 검색
- 파일 버전 관리
- 로그인 인증

# 4. 기술 관리 계획
## 4.1 버전 관리
소프트웨어는 파일명 맨 앞에 버전을 기재한다.\
문서는 파일명 맨 뒤에 날짜를 기재한다.\
두가지 모두 Github에 저장한다.

## 4.2 형상 관리
소프트웨어의 경우 버전_이름의 형식으로 명명한다.\
문서의 경우 문서명_날짜의 형식으로 명명한다.

## 4.3 기술 관리
기능 변경시 팀장의 확인이 필요하며, 변경 전후 내용을 기록물로 보존한다.

# 5. 품질 관리 계획
## 5.1 품질 목표
| 품질 특성 | 구체적 목표 |
| --- | --- |
| 기능 적합성 | 파일 업로드/다운로드, 공유, 권한 설정 등 핵심 기능이 정확하게 동작해야 함. |
| 성능 효율성 | 검색과 다운로드 기능이 빠르게 동작해야함. 200명 정도의 동시 사용에도 안정적으로 동작해야함. |
| 사용성 | 버튼을 누르고 파일을 선택하는 정도로 업로드 방식이 간단해야함. 휴지통 기능으로 파일 복구가 가능해야함. |
| 신뢰성 | 업무 시간 중 가용성이 99.9% 이상 유지되어야 하며 서버 장애 시에도 데이터가 유실되지 않아야 함. |
| 보안성 | 데이터 암호화, 로그인으로 사용자 인증 |
| 유지보수성 | 향후 필요할 가능성이 있는 문서 공동 편집, 파일 자동 분류 등의 기능을 추가하기 쉬운 구조여야 함. |
| 이식성 | Chrome, Safari, Edge 등의 여러 브라우저에서 작동해야 함. |

## 5.2 검토 방법 및 주기
프로젝트 리더가 진행 상황 확인 시 요구 사항의 반영 현황도 확인한다.\
V 모델의 테스트 단계를 따라 최종 검토를 진행한다.

## 5.3 품질 관리 적용 기법
척도를 마련하고 데이터를 측정해 품질 수준을 평가한다.

## 5.4 리스크 관리
팀원 변경 시 일시적으로 작업량을 낮추고 새 팀원을 구한다.
