# 요구사항 정의서

## 1. 개요
### 1.1 문서의 목적
이 문서는 파일 공유 시스템을 개발하는 프로젝트에서 각 요구 사항이 무엇인지 조사하고, 정의하는 문서이다.  
이 문서는 기능적, 비기능적, 인터페이스에 요구되는 사항들을 정의한다.

### 1.2 프로젝트 개요
업무 문서 공유를 위한 클라우드 기반 파일 공유 시스템을 개발한다.

## 2. 요구사항
### 2.1 기능적 요구사항

| F-005 | 폴더 | 사용자는 폴더의 이름을 변경한다. | medium |
| F-006 | 폴더 | 사용자는 폴더를 삭제한다. | high |
| F-007 | 폴더 | 사용자는 파일을 다른 폴더로 이동한다. | high |
| F-008 | 파일 저장 | 사용자는 30일 이내 삭제된 파일을 복구한다. | medium |

| F-010 | 파일 공유 | 사용자는 파일 공유 시 공유 대상자의 권한을 열람, 수정, 댓글 중에서 설정한다. | high |

| F-013 | 파알 공유 | 사용자는 공유 링크의 만료 기간을 설정한다. | high |
| F-014 | 파일 검색 | 사용자는 파일 이름으로 파일을 검색한다. | high |
| F-015 | 파일 검색 | 사용자는 파일의 업로드 날짜, 업로드한 사용자, 파일 유형으로 검색을 필터링한다. | medium |
| F-016 | 파일 버전 관리 | 시스템은 파일의 수정 시 최근 10개의 이전 버전을 보관한다. | high |
| F-017 | 파일 버전 관리 | 사용자는 파일의 이전 버전 목록을 확인한다. | high |
| F-018 | 파일 버전 관리 | 사용자는 파일의 이전 버전을 다운로드한다. | high |
| F-019 | 파일 버전 관리 | 사용자는 파일을 이전 버전으로 복원한다. | high |
| F-020 | 계정 | 관리자는 사용자 계정을 생성한다. | high |
| F-021 | 계정 | 관리자는 사용자 계정을 삭제한다. | high |

| F-023 | 계정 | 관리자는 사용자 그룹을 생성한다. | medium |
| F-024 | 계정 | 사용자는 이메일과 비밀번호를 입력하여 로그인한다. | high |
